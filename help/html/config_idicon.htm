Identity Icon Selection

Starting with Network Identity Manager 2.0, you can customize the icon displayed next to an identity. This dialog lets you pick an image of your choice from your computer, or from a web page.

Identity icon selection dialog

There are currently four ways to specify an image file to be used as the basis of your icon:

  • Use picture from file: If the image is already on your computer, then you can use the Browse button to locate the file. Once selected, the image will be loaded into the area on the left.

  • Fetch from URL: Fetches an image from the internet. Currently, only http:// URLs are accepted and the URL must locate the image directly. Once the URL is entered, click on the Go button to retrieve the image into the crop area

  • Favicon for domain: Attempts to fetch the favicon for a domain. If the identity is associated with a domain, it will be pre-filled for you.

    If the listed domain doesn't have a favicon, the application will attempt to fetch the favicon for the parent domains of the specified domain.

  • Gravatar for email address: Attempts to download the Gravatar corresponding to the specified email address. If a likely email address is already associated with the identity, it will be pre-filled for you.

Once you specify the image file to use, NetIDMgr will load the image into the crop area on the left side of the dialog. You can drag the borders of the crop rectangle to select the portion of the image you would like to use as your icon.

When you are satisfied with your new icon, you can click the Ok button to set the new icon.

Supported Image Formats

The following image formats are currently supported:

  • BMP: Windows bitmap files
  • JPG
  • PNG
  • ICO: Windows icon files
  • EXE,DLL: Icon and bitmap resources embedded in .EXE and .DLL files can also be used.

Privacy and Security Issues with fetching identity icons from the web

All three methods of fetching the identity icon from the web outlined above raise privacy and security concerns. These are detailed in the Privacy and Security Issues page.
