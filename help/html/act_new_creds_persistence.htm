Password Persistence

Starting with Network Identity Manager 2.0, you have the option of securely storing your passwords using the included KeyStore plug-in.

The utility of storing passwords is most apparent if you have more than one identity that you obtain initial credentials for at once. When you acquire credentials for a key store, Network Identity Manager automatically initializes all the identities that are stored there. Therefore, with a single password you can obtain credentials for multiple identities.

Saving your password

The password entry pages in the new credentials wizard will ask you whether you would like to save the entered password. Simply check the option. You will then be asked for the master password the KeyStore. If you haven't set up a master password, it will ask you to enter the password twice, since you are setting up a new password. Once the dialog completes, your password will be added to the key store.

Obtaining new credentials using the key store

Select an identity that is stored in the key store and invoke the New credentials wizard. You will automatically be prompted for the key store credentials.

After you supply the key store master password, credentials for all the identities stored in the key store will be obtained.

Removing stored passwords from a key store

Start the configuration dialog and select the key store identity. (Alternatively, right-click the key store identity in the credentials display and select show identity options).

In the identity configuration page for the key store, select the Keystore tab.

Select the identity you want to remove the stored password for, and select Remove.

If you remove the configuration information for an identity using the configuration dialog, any passwords stored for that identity will be automatically removed.
