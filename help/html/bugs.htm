Reporting a Problem

If you encounter a problem in the software, please send a bug report via e-mail to netidmgr@secure-endpoints.com. Please include as much information as possible to assist us in reproducing the problem.

kerberos@MIT.EDU is a mailing list set up for discussing end user Kerberos issues. It is gatewayed to the Usenet newsgroup 'comp.protocols.kerberos'. If you prefer to read it via mail, send a subscription request to kerberos-request@MIT.EDU or subscribe via the web page:

http://mailman.mit.edu/mailman/listinfo/kerberos

Information about Kerberos mailing lists can be found at http://web.mit.edu/kerberos/mail-lists.html

What to include in a bug report

Please include the following information in a bug report:

  • Explanation of the problem.

  • Version of Network Identity Manager. This can be determined by selecting Help -> About Network Identity Manager from the main menu.

  • List of plug-ins that are installed, other than the defaults. By default, Network Identity Manager version 2.0 includes the following plug-ins: KeyStoreIdentity, KeyStoreCred, Krb5Ident, Krb5Cred and on 32-bit systems Krb4Cred. Please note if any other plug-ins are installed on your machine and the listed version for each.

    The installed plug-ins can be inspected by selecting Options -> Plug-ins from the main menu.

  • If the problem occurs during a credentials acquisition or other credentials operation, also attempt to capture a trace log by doing the following:

    Log trace events option in the
    General Options configuration panel.

    1. Select Options -> General from the main menu to open the General options page.

    2. Check the Log trace events to the trace log... option.

    3. Attempt to reproduce the problem.

    4. Open the General options page again, and click on the Show log button.

    5. Include the contents of the captured log in the problem report.

    6. Once you have captured the trace information, you may disable trace logging.
